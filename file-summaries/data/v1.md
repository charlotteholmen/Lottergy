# data/v1/

**What:** The public data contract — one JSON file per supported game, refreshed nightly by `.github/workflows/scrape.yml`.

**Why:** GOAL.md locked decision: data hosted in the same repo, served via GitHub Pages, refreshed nightly. v1 is path-versioned (`/v1/`); breaking schema changes will bump to `/v2/`.

## Files

- `powerball.json` — written by `scripts/scraper/powerball.ts`. Currently ~1.9k draws back to 2010.
- `megamillions.json` — written by `scripts/scraper/megamillions.ts`. Currently ~2.5k draws back to 2002.

## Schema (matches CLAUDE.md data contract verbatim)

```json
{
  "game": "powerball",
  "updated_at": "2026-05-22T08:00:00Z",
  "matrix": { "whites_max": 69, "reds_max": 26, "whites_per_ticket": 5 },
  "eras": [{ "start": "2015-10-07", "whites_max": 69, "reds_max": 26, "label": "5/69 + 1/26" }],
  "draws": [{ "date": "2026-05-20", "whites": [3, 14, 27, 41, 62], "red": 11 }]
}
```

## Key invariants

- Draws sorted ascending by date; whites within each draw sorted ascending.
- `matrix` reflects the CURRENT era. The full era timeline lives in `eras`.
- `updated_at` is the scraper-run timestamp, NOT the latest draw date.
- Do not hand-edit these files. They are scraper output; the scraper is the source of truth.
- File size budget: ~1MB/game is the soft cap. Past that, switch to dated-shards or trim with a date floor.
